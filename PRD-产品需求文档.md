## 一、产品目标

### 1.1 核心价值
一款**基于 AI 大模型的职场高情商话术实时生成工具**，用户输入具体情境描述，AI 秒级生成得体、自然、不卑不亢的回复话术，支持 4 种语气风格切换，帮助职场人轻松应对各类敏感沟通场景。

### 1.2 参赛定位
- **赛道**：More Than Coding（MTC）+ Code 双赛道
- **目标**：打造一款"打开即用、用完即走、忍不住分享"的轻量 AI 工具
- **冲榜策略**：全人群覆盖 × 强刚需 × 高互动 × 轻量传播
- **截止日期**：2025 年 4 月 26 日（剩余 12 天）

### 1.3 产品 Slogan
> 告别尴尬，AI 帮你说话

## 二、目标用户

### 2.1 核心用户群
全职场人群，不分岗位、不分技术/非技术背景。

### 2.2 细分场景用户

| 用户画像 | 典型场景 | 核心诉求 |
|---------|---------|---------|
| 职场新人（0-3 年） | 不知道怎么拒绝、怎么汇报 | 降低沟通门槛，避免犯错 |
| 中层骨干（3-7 年） | 谈薪、跨部门协作、绩效沟通 | 专业得体，维护自身权益 |
| 管理者/Leader | 向上汇报、团队管理沟通 | 高效表达，树立威信 |
| 求职者 | 与 HR/猎头沟通、拒 offer | 话术专业，不吃亏 |
| 即将离职者 | 离职告别、交接沟通 | 好聚好散，不留遗憾 |

| 风格 | 适用对象 | 核心特征 | 示例关键词 |

## 五、交互流程

### 5.1 主流程（首次使用）
```
打开网页
  → 展示每日金句（顶部横幅）
  → 用户看到 8 个场景按钮 + 输入框
  → [路径 A] 点击预设场景 → 输入框自动填充引导文案 → 用户修改补充
  → [路径 B] 直接在输入框描述自己的情境
  → 选择语气风格（默认温和）
  → 点击「生成话术」按钮
  → loading 动画（"AI 正在为你组织语言..."）
  → 打字机效果逐字展示话术
  → 用户可：一键复制 / 重新生成 / 切换语气 / 点赞
  → 底部引导：分享 / 提交场景需求
```

### 5.2 回访流程
```
打开网页
  → 展示新的每日金句
  → 热门场景 TOP3 排行（激发好奇心）
  → 用户尝试热门场景或探索新场景
  → 使用后可提交自己的场景需求
  → 场景征集提交成功 → 弹出感谢提示
```

### 5.3 异常流程
```
用户点击生成
  → API 调用失败（网络错误/额度用完）
  → 自动降级到预设话术库
  → 展示预设话术 + 提示"当前使用离线话术库"
  → 用户仍可复制使用
```

### 5.4 评审补充：体验优化点
1. **首次进入引导**：输入框 placeholder 要足够吸引人，降低使用门槛
2. **空状态处理**：未生成话术时，展示区显示引导文案而非空白
3. **生成中状态**：按钮置灰，防止重复点击
4. **复制反馈**：复制成功后按钮文字变为"已复制 ✓"，2 秒后恢复
5. **语气切换联动**：切换语气后自动重新生成，无需手动再点生成

## 八、风险与优化点

### 8.1 已识别风险

| 风险 | 概率 | 影响 | 应对方案 |
|------|------|------|---------|
| MiniMax API 额度用完 | 低 | 无法 AI 生成 | 预设话术降级方案，每个场景 3-5 条 |
| API 响应慢/超时 | 中 | 用户体验差 | loading 动画 + 5 秒超时自动降级 |
| 前端 API Key 被滥用 | 中 | 额度消耗 | Base64 编码 + 前端限流（每人每天 20 次） |
| 手机端兼容问题 | 低 | 部分功能异常 | 优先测试 iOS Safari + 微信浏览器 |
| 话术质量不稳定 | 中 | 用户信任度下降 | Prompt 精调 + 生成约束 + 温馨提示 |

### 8.2 优化方向（评审补充）

| 优化点 | 说明 | 优先级 |
|--------|------|--------|
| **话术质量是生命线** | AI 生成的话术必须自然、得体、可用。Prompt 需要反复调优，建议开发阶段人工测试每个场景 × 每种语气 | P0 |
| **首次体验决定留存** | 用户第一次使用必须 3 步内看到结果：选场景 → 看到引导 → 一键生成 | P0 |
| **传播裂变设计** | 生成的话术要自带"分享欲"——有趣、有用、有共鸣 | P1 |
| **数据可视化** | 热门排行用简单的进度条/数字展示，比纯文字更有冲击力 | P2 |
| **空状态设计** | 未生成话术时的引导区要精心设计，不能是空白 | P1 |
